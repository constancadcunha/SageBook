🎯 Evaluation Goals

The evaluation plan is built around validating four core objectives of the SageBook app:

Objective Evaluation Focus
1. Improve Culinary Confidence Does SageBook reduce intimidation for beginners and help users feel capable in the kitchen?
2. Teach Foundational Skills Are users retaining skills from tutorials and applying them in recipes?
3. Motivate Continued Use Do gamification and progress tracking encourage return visits and ongoing learning?
4. Support Seamless Cooking Is the app practical and usable during real-time kitchen activity (hands dirty, multitasking)?

Each goal will be supported by multiple qualitative and quantitative research methods.

🔬 Research & Evaluation Methods

Method Purpose Timeline
Usability Testing Evaluate ease of navigation, clarity of recipe steps, and interface responsiveness Pre-launch / mid-fi and hi-fi prototype stage
Heuristic Evaluation Identify usability issues through expert review based on design principles Pre-launch
Think-Aloud Protocol Understand user cognitive processes as they complete tasks in the app During usability testing
Task Success Metrics Track whether users complete specific flows (e.g., finish a recipe, add to shopping list) Ongoing
System Usability Scale (SUS) Quantitative usability benchmark Pre-launch and post-launch
Diary Studies Assess real-world use during actual cooking over several days Beta testing phase
In-App Analytics Monitor feature usage (tutorials accessed, recipes completed, streaks maintained) Post-launch
Surveys (Pre/Post-Use) Measure changes in confidence, skill, and satisfaction Ongoing
A/B Testing Optimize specific features like tip visibility or gamification cues Post-launch

🧭 User Testing Plan

🎯 Target Users

  • Beginners (Primary Audience): Young adults, college students, or new home cooks
  • Intermediate Hobbyists: Home bakers and cooks looking to level up
  • Dietary Explorers: Users looking for substitutions or with specific food preferences

🧪 Usability Testing: Scenarios & Tasks

Scenario Task Success Criteria
"I'm trying a recipe after work." Find a pasta recipe, follow all steps, and complete it Finishes within expected time, uses tips without confusion
"I need a dairy-free option." Search for or substitute a recipe ingredient Finds appropriate substitution, adds to shopping list
"I'm learning to knead dough." Navigate to skill library, find 'kneading dough', watch tutorial Watches video, can recall/perform key steps later
"I want to improve my baking streak." Use gamified dashboard, start a new recipe Understands how streaks work, is motivated to continue
"I forgot how to brown butter." Mid-recipe, access tip/help from a step Uses embedded tip or skill pop-up successfully

📊 Quantitative Metrics to Track

Metric Why It Matters
Task Completion Rate Are users able to complete key flows without getting stuck?
Time on Task Is navigation intuitive and efficient?
SUS Score (Goal: 80+) Benchmark of perceived usability
Feature Engagement Rate Tutorials accessed, badges earned, community tips viewed
Retention Rate (Day 1, 7, 30) Are users coming back to continue learning?
Streak Maintenance Rate Effectiveness of gamification in driving habits
Help/Tip Usage Frequency Are embedded learning supports used and helpful?
Shopping List Conversions Do users add and use generated lists (real-world utility)?

🧠 Qualitative Insights to Gather

Research Focus Questions to Explore
Clarity of Instructions "Did you ever feel confused about what to do next?"
Confidence Building "Do you feel more capable in the kitchen after using the app?"
Emotional Resonance "How would you describe how the app feels to use?"
Motivation Factors "What made you want to keep using the app — or not?"
Accessibility in Real-Use Context "Was it easy to use while actively cooking, or did you run into barriers?"
Tutorial Usefulness "Did the skill tutorials feel valuable? Would you use them again?"

🧩 Validation of Key Features

Feature How It Will Be Evaluated
Step-by-Step Recipes Observational testing + think-aloud to assess clarity and flow
Gamification (badges, streaks) A/B test with and without cues + diary feedback on motivation
Skill Tutorials Measure tutorial access and post-use confidence/success in application
Ingredient Substitutions Real-time task tests + survey on dietary flexibility
Community Tips Monitor usage patterns + interview users for perceived value
Shopping List Integration Track list usage, additions, and kitchen conversions
Visual/Audio Prompts Accessibility feedback + situational testing (hands full, distractions)
